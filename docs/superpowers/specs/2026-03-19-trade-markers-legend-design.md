# 持仓/自选基金折线图交易点与图例设计

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 优化持仓与自选基金详情页折线图的交易点展示，改为红/蓝/黄圆点，并在图表上方展示买卖/锚点图例。

**Non-Goals:** 不新增数据源、不改动现有数据结构、不改变折线图性能策略与区间切换逻辑。

**Constraints:** 遵循现有 ECharts + Tailwind 结构，不引入新的图表库与全局样式。

---

## 背景与问题

- 当前图表使用蓝色 pin 形状显示买点/锚点，视觉与交易点语义不匹配。
- 需要清晰区分买入、卖出与清仓，并提供图例说明，避免误读。

## 设计目标

- 所有交易点以圆点显示，颜色表达语义：买入红、卖出蓝、清仓黄。
- 不再显示蓝色气泡状的买点与锚点（pin + 文本）。
- 在折线图表上方展示图例，清楚标识三种点。
- 兼容持仓与自选详情页，保持暗色模式可读性。

## 关键规则

### 交易点颜色规则

- 买入点：红色圆点。
- 卖出点：蓝色圆点。
- 清仓点：黄色圆点。

### 清仓点判定

- 规则：卖出交易后持仓为 0 的交易日标黄点。
- 判定方式：仅在 `fund.holdingShares === 0` 且存在一笔卖出交易可对应到该日期时才标黄点。
- 若无法可靠判定持仓为 0，则不绘制清仓点，只绘制买/卖点。

## 数据来源与逻辑

- 交易点数据继续使用现有字段：
  - `fund.buyDate`
  - `fund.pendingTransactions`
- 不引入新 API、不新增持久化字段。
- 清仓点通过交易序列计算持仓变化；仅在可推导到 0 时标记黄点。

## 交互与视觉

### 点样式

- `symbol: 'circle'`。
- `symbolSize: 8`。
- 仅显示点，不显示文字标签。
- 不使用 `pin`、`arrow`、`triangle` 形状。
- 颜色取值（与现有主题风格一致）：
  - 买入红：`#f87171`
  - 卖出/锚点蓝：`#3b82f6`
  - 清仓黄：`#fbbf24`

### 图例

- 在折线图容器顶部，位于日期标题行下方（同一卡片区域内）。
- 图例为纯文本 + 小圆点（与点颜色一致）。
- 图例固定显示，不随缩放隐藏。
- 图例文案随上下文变化：
  - 持仓详情：`买入` / `卖出` / `清仓`
  - 自选详情：`锚点`（仅显示蓝点标签）

### 暗色模式

- 颜色在暗色背景下保持足够对比度。
- 文本颜色随主题（现有 `isDark`）适配。

## 适用范围

- 持仓详情页：以买/卖/清仓点表达交易行为。
- 自选详情页：保留锚点语义，使用蓝色圆点表达锚点，不再使用 pin。
- 不修改其他图表（例如指数/市场页）。

## 受影响文件

- `components/FundDetail.tsx`：
  - `markPoint` 配置替换为圆点样式。
  - 新增图表上方的图例 DOM。
  - 清仓点计算逻辑（基于交易序列）。

## 风险与边界

- 交易序列缺少完整持仓变化时，清仓点可能无法准确判定，需保证不误标。
- 仅改样式与可视化标注，不更改收益/成本计算逻辑。

## 验收标准

- 买入点显示为红色圆点。
- 卖出点显示为蓝色圆点。
- 清仓点显示为黄色圆点（仅在可判定时）。
- 不再出现蓝色 pin 形状的买点或锚点。
- 图表上方显示点图例：持仓页显示买/卖/清仓，自选页显示锚点。
- 现有 tooltip 行为保持不变。
